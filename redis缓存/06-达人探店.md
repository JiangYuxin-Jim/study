# 06 - 达人探店

## 1. 发布探店笔记

### 1.1 业务场景

用户（达人）在平台上发布探店笔记，包含文字描述、探店图片、探店位置等信息。

```
用户 ──→ [编辑笔记] ──→ [上传图片] ──→ [提交发布] ──→ [保存到MySQL]
```

### 1.2 数据表设计

```sql
-- 探店笔记表
CREATE TABLE tb_blog (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    shop_id     BIGINT COMMENT '关联商铺ID',
    user_id     BIGINT COMMENT '发布者用户ID',
    title       VARCHAR(255) COMMENT '笔记标题',
    images      VARCHAR(2000) COMMENT '图片列表（多张图用逗号分隔）',
    content     TEXT COMMENT '笔记正文',
    liked       INT DEFAULT 0 COMMENT '点赞数量',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 1.3 图片上传

图片上传到本地服务器（Nginx 静态资源目录），返回访问 URL 存入数据库：

```java
@PostMapping("/blog/upload")
public Result uploadImage(@RequestParam("file") MultipartFile image) {
    // 1. 生成唯一文件名（UUID + 原始后缀）
    String originalFilename = image.getOriginalFilename();
    String suffix = originalFilename.substring(originalFilename.lastIndexOf("."));
    String fileName = UUID.randomUUID().toString() + suffix;

    // 2. 创建存储目录（不存在则创建）
    File dir = new File(SystemConstants.IMAGE_UPLOAD_DIR);
    if (!dir.exists()) {
        dir.mkdirs();
    }

    // 3. 保存图片到 Nginx 静态资源目录
    image.transferTo(new File(dir, fileName));

    // 4. 返回可访问的图片 URL
    return Result.ok("/imgs/" + fileName);
}
```

### 1.4 发布笔记完整代码

```java
@PostMapping("/blog")
public Result saveBlog(@RequestBody Blog blog) {
    // 获取当前登录用户
    Long userId = UserHolder.getUser().getId();
    blog.setUserId(userId);
    // 保存到数据库
    blogService.save(blog);
    // 返回笔记ID
    return Result.ok(blog.getId());
}
```

## 2. 查看探店笔记

### 2.1 笔记列表 / 详情查询

```java
// 查询笔记详情（含发布者信息）
@GetMapping("/blog/{id}")
public Result queryBlogById(@PathVariable Long id) {
    Blog blog = blogService.getById(id);
    if (blog == null) {
        return Result.fail("笔记不存在");
    }
    // 查询发布者信息
    User user = userService.getById(blog.getUserId());
    blog.setNickName(user.getNickName());
    blog.setIcon(user.getIcon());
    // 判断当前用户是否已点赞
    isBlogLiked(blog);
    return Result.ok(blog);
}
```

### 2.2 分页查询热门笔记

```java
@GetMapping("/blog/hot")
public Result queryHotBlog(@RequestParam(value = "current", defaultValue = "1") Integer current) {
    Page<Blog> page = blogService.query()
            .orderByDesc("liked")  // 按点赞数降序
            .page(new Page<>(current, SystemConstants.MAX_PAGE_SIZE));
    // 获取当前页数据
    List<Blog> records = page.getRecords();
    return Result.ok(records);
}
```

## 3. 点赞功能 — 基于 Redis Set

### 3.1 为什么用 Redis 而不是 MySQL？

| | MySQL 方案 | Redis 方案 |
|------|-----------|-----------|
| **写入速度** | 每次点赞写一行 `tb_blog_like`，高频下压力大 | `SADD` 内存操作，微秒级 |
| **查询速度** | `SELECT COUNT(*)` 全表扫描 | `SCARD` O(1) |
| **去重判断** | 再查一次 DB | `SISMEMBER` O(1) |
| **并发安全** | 需要加锁 | Redis 单线程，天然原子 |

### 3.2 数据结构选型

```
Redis Set 天然去重，适合存储"谁点了赞"：

Key:   liked:blog:{blogId}
Type:  Set
Value: {userId1, userId2, userId3, ...}
```

| 操作 | Redis 命令 | 含义 |
|------|-----------|------|
| 点赞 | `SADD liked:blog:{blogId} {userId}` | 用户加入点赞集合 |
| 取消点赞 | `SREM liked:blog:{blogId} {userId}` | 用户从集合移除 |
| 判断是否点赞 | `SISMEMBER liked:blog:{blogId} {userId}` | 返回 1 表示已点赞 |
| 获取点赞数 | `SCARD liked:blog:{blogId}` | 返回集合元素数量 |

### 3.3 点赞 / 取消点赞代码

```java
@PutMapping("/blog/like/{id}")
public Result likeBlog(@PathVariable Long id) {
    Long userId = UserHolder.getUser().getId();
    String key = "liked:blog:" + id;

    // 判断当前用户是否已点赞
    Boolean isLiked = stringRedisTemplate.opsForSet().isMember(key, userId.toString());

    if (Boolean.TRUE.equals(isLiked)) {
        // 已点赞 → 取消点赞
        stringRedisTemplate.opsForSet().remove(key, userId.toString());
        // 同步更新数据库中的点赞数
        blogService.update().setSql("liked = liked - 1").eq("id", id).update();
    } else {
        // 未点赞 → 点赞
        stringRedisTemplate.opsForSet().add(key, userId.toString());
        blogService.update().setSql("liked = liked + 1").eq("id", id).update();
    }

    return Result.ok();
}
```

### 3.4 查询时判断登录用户是否已点赞

```java
private void isBlogLiked(Blog blog) {
    UserDTO user = UserHolder.getUser();
    if (user == null) {
        // 未登录，不需判断
        return;
    }
    String key = "liked:blog:" + blog.getId();
    Boolean isLiked = stringRedisTemplate.opsForSet()
        .isMember(key, user.getId().toString());
    blog.setIsLike(Boolean.TRUE.equals(isLiked));
}
```

## 4. 点赞排行榜 — 基于 Redis SortedSet

### 4.1 新需求：Top N 点赞用户按时间排序

Set 只能存"谁点了赞"，无法按**点赞先后时间排序**。产品要求展示"最早点赞的 Top 5 用户"。

```
SortedSet 结构适用于此场景：

Key:    liked:blog:top:{blogId}
Score:  点赞时间戳（毫秒）
Member: 用户ID

ZADD 加入时按时间戳自动排序，ZRANGE 直接取 Top N
```

### 4.2 SortedSet 常用命令

| 命令 | 说明 |
|------|------|
| `ZADD key score member` | 添加元素（score 用于排序） |
| `ZREM key member` | 移除元素 |
| `ZSCORE key member` | 获取元素的 score（NULL 表示不存在） |
| `ZRANGE key 0 4` | 返回 score 最小的前 5 个元素（升序） |
| `ZRANGE key 0 4 WITHSCORES` | 连 score 一起返回 |
| `ZCARD key` | 获取元素个数 |
| `ZPOPMIN key count` | 弹出 score 最小的 N 个元素 |

### 4.3 代码实现

```java
@PutMapping("/blog/like/{id}")
public Result likeBlog(@PathVariable Long id) {
    Long userId = UserHolder.getUser().getId();
    String key = "liked:blog:top:" + id;

    // ZSCORE 判断用户是否已点赞（替代 Set 的 SISMEMBER）
    Double score = stringRedisTemplate.opsForZSet().score(key, userId.toString());

    if (score != null) {
        // 已点赞 → 取消点赞
        stringRedisTemplate.opsForZSet().remove(key, userId.toString());
        blogService.update().setSql("liked = liked - 1").eq("id", id).update();
    } else {
        // 未点赞 → 点赞（score = 当前时间戳）
        stringRedisTemplate.opsForZSet()
            .add(key, userId.toString(), System.currentTimeMillis());
        blogService.update().setSql("liked = liked + 1").eq("id", id).update();
    }

    return Result.ok();
}
```

### 4.4 查询点赞排行榜 Top 5

```java
@GetMapping("/blog/likes/{id}")
public Result queryBlogLikes(@PathVariable Long id) {
    String key = "liked:blog:top:" + id;

    // 按 score 升序取前 5 个（最早点赞的排最前）
    Set<String> top5UserIds = stringRedisTemplate.opsForZSet()
        .range(key, 0, 4);

    if (top5UserIds == null || top5UserIds.isEmpty()) {
        return Result.ok(Collections.emptyList());
    }

    // 查询用户详细信息（注意 MySQL 的 IN 查询不保证顺序）
    List<Long> ids = top5UserIds.stream().map(Long::valueOf).collect(Collectors.toList());
    List<UserDTO> users = userService.listByIds(ids);

    // 按 ZSET 的排序还原顺序（MySQL IN 查询会打乱）
    Map<Long, UserDTO> userMap = users.stream()
        .collect(Collectors.toMap(UserDTO::getId, u -> u));
    List<UserDTO> sortedUsers = ids.stream()
        .map(userMap::get)
        .collect(Collectors.toList());

    return Result.ok(sortedUsers);
}
```

> **关键点**：MySQL 的 `WHERE id IN (...)` 不保证返回顺序和 IN 列表一致，需要在 Java 层手动还原 ZSET 返回的排序。

### 4.5 Set vs SortedSet 对比

| | Set | SortedSet |
|------|-----|-----------|
| **底层结构** | Hash 表 | 跳表 + Hash 表 |
| **是否有序** | ❌ 无序 | ✅ 按 score 排序 |
| **去重** | ✅ 天然去重 | ✅ 天然去重 |
| **判断成员存在** | `SISMEMBER` O(1) | `ZSCORE` O(log N) |
| **获取点赞数** | `SCARD` O(1) | `ZCARD` O(1) |
| **获取排行** | ❌ 不支持 | ✅ `ZRANGE` |
| **适用场景** | 只需知道"谁赞了" | 需要"按时间/热度排行" |
| **本场景推荐** | ⚠️ 早期方案 | ✅ **最终方案** |

## 6. 总结

| 知识点 | 核心要点 |
|--------|----------|
| 图片上传 | UUID 生成唯一文件名，保存到 Nginx 静态目录 |
| 发布笔记 | 包含文字+图片 URL，存入 MySQL `tb_blog` 表 |
| 查看笔记 | 分页查热门（按 `liked` 降序），详情含发布者信息 |
| 为什么用 Redis 做点赞 | 写入快、去重天然支持、判断 O(1)、缓解 MySQL 压力 |
| Redis Set 点赞 | `SADD`/`SREM`/`SISMEMBER`/`SCARD`，简单去重集合 |
| Set 方案的局限 | 无序，无法按点赞时间排行 |
| Redis SortedSet | `ZADD`(score=时间戳)/`ZSCORE`(判断)/`ZRANGE`(排行)/`ZREM` |
| 排行榜 Top 5 | `ZRANGE key 0 4` 取前 5，注意 MySQL IN 查询顺序需手动还原 |
| 点赞/取消点赞 | `ZSCORE` 判 NULL → 赞/取消赞，同步更新 MySQL `liked ± 1` |
| Set vs SortedSet | Set 适合"谁赞了"，SortedSet 适合"按时间/热度排行" |
