# 08 - 员工管理实战（上）：新增、事务、文件上传

## 1. 新增员工

### 1.1 接口

```
POST /emps
{
    "username": "zhangsan",
    "name": "张三",
    "gender": 1,
    "deptId": 2,
    "salary": 8000
}
```

### 1.2 实现

```java
// Controller
@PostMapping
public Result add(@RequestBody Emp emp) {
    empService.add(emp);
    return Result.success();
}

// Service
@Override
public void add(Emp emp) {
    emp.setCreateTime(LocalDateTime.now());
    emp.setUpdateTime(LocalDateTime.now());
    empMapper.insert(emp);
}

// Mapper
@Insert("INSERT INTO tb_emp(username, name, gender, dept_id, salary, create_time, update_time) " +
        "VALUES(#{username}, #{name}, #{gender}, #{deptId}, #{salary}, #{createTime}, #{updateTime})")
void insert(Emp emp);
```

## 2. 事务管理

### 2.1 问题场景

新增员工时，除了写入 `tb_emp`，可能还要更新 `tb_dept`（部门人数 +1）。如果第二步失败了，第一步已插入的数据就成了脏数据。

```
步骤1：INSERT INTO tb_emp ✓ （成功）
步骤2：UPDATE tb_dept SET emp_count = emp_count + 1 ✗ （失败）
→ 结果：员工记录了但部门人数没变 → 数据不一致！
```

### 2.2 事务四大特性（ACID）

| 特性 | 说明 |
|------|------|
| **原子性 Atomicity** | 所有操作要么全成功，要么全失败（回滚） |
| **一致性 Consistency** | 事务前后数据状态保持一致 |
| **隔离性 Isolation** | 并发事务之间互不干扰 |
| **持久性 Durability** | 事务提交后数据永久保存 |

### 2.3 Spring 事务管理 — @Transactional

```java
@Service
public class EmpServiceImpl implements EmpService {

    @Autowired
    private EmpMapper empMapper;
    @Autowired
    private DeptMapper deptMapper;

    @Override
    @Transactional  // 方法内所有 DB 操作要么全成功，要么全回滚
    public void add(Emp emp) {
        emp.setCreateTime(LocalDateTime.now());
        empMapper.insert(emp);                           // 步骤1
        deptMapper.incrementCount(emp.getDeptId());      // 步骤2
        // 如果步骤2抛异常 → 步骤1自动回滚
    }
}
```

**@Transactional 常见属性**：

```java
@Transactional(
    propagation = Propagation.REQUIRED,  // 事务传播行为（默认：有则加入，无则新建）
    rollbackFor = Exception.class        // 遇到什么异常回滚（默认：RuntimeException + Error）
)
```

### 2.4 事务传播行为

| 传播行为 | 说明 |
|----------|------|
| `REQUIRED`（默认） | 有事务就加入，没有就新建 → **最常用** |
| `REQUIRES_NEW` | 总是新建事务，挂起当前事务 |
| `SUPPORTS` | 有事务就加入，没有就算了（非事务运行） |
| `NOT_SUPPORTED` | 总是非事务运行，挂起当前事务 |

**案例**：记录操作日志时使用 `REQUIRES_NEW`，日志记录失败不影响主业务：

```java
@Transactional
public void add(Emp emp) {
    empMapper.insert(emp);               // 主事务
    logService.record("新增员工");       // 日志独立事务（失败不影响主事务）
}

@Service
class LogService {
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void record(String action) { ... }
}
```

## 3. 文件上传

### 3.1 前端上传

```html
<form enctype="multipart/form-data">
    <input type="file" name="image">
    <button type="submit">上传</button>
</form>
```

### 3.2 后端接收

```java
@PostMapping("/upload")
public Result upload(@RequestParam("image") MultipartFile file) throws IOException {
    // 1. 获取原始文件名和后缀
    String originalFilename = file.getOriginalFilename();
    String suffix = originalFilename.substring(originalFilename.lastIndexOf("."));

    // 2. 生成唯一文件名（UUID + 后缀）
    String newFileName = UUID.randomUUID().toString() + suffix;

    // 3. 保存到本地磁盘
    String filePath = "E:/upload/" + newFileName;
    file.transferTo(new File(filePath));

    // 4. 返回访问 URL
    return Result.success("/files/" + newFileName);
}
```

**配置上传限制**（application.properties）：

```properties
# 单个文件最大大小
spring.servlet.multipart.max-file-size=10MB
# 单次请求最大大小
spring.servlet.multipart.max-request-size=100MB
```

### 3.3 阿里云 OSS（对象存储）

生产环境中，文件应存储在独立的文件服务器或云存储（OSS）上，而不是应用服务器本地。

```
用户 ──→ [SpringBoot] ──→ [阿里云 OSS]
        ① 上传文件           ② 返回 URL
        ③ 把 URL 存入数据库
```

```java
// 使用阿里云 OSS SDK
@PostMapping("/upload")
public Result upload(MultipartFile file) throws IOException {
    // 1. 生成唯一文件名
    String fileName = UUID.randomUUID() + file.getOriginalFilename();

    // 2. 上传到 OSS
    OSS ossClient = new OSSClientBuilder().build(endpoint, accessKeyId, accessKeySecret);
    ossClient.putObject(bucketName, fileName, file.getInputStream());
    ossClient.shutdown();

    // 3. 拼接访问 URL
    String url = "https://" + bucketName + "." + endpoint + "/" + fileName;
    return Result.success(url);
}
```

> **好处**：不占用应用服务器磁盘，OSS 自带 CDN 加速、高可用、无限扩容。

## 4. 总结

| 知识点 | 核心要点 |
|--------|----------|
| 新增员工 | Controller 接收 JSON，Service 设时间戳，Mapper Insert |
| ACID | 原子性（全成功/全失败）、一致性、隔离性、持久性 |
| `@Transactional` | 声明式事务，方法内所有 DB 操作自动纳入事务管理 |
| 传播行为 | `REQUIRED`(默认) `REQUIRES_NEW`(日志等独立事务) |
| 文件上传 | `MultipartFile` 接收，`transferTo` 保存到本地 |
| OSS | 对象存储，解耦文件和应用服务器，自带 CDN |
