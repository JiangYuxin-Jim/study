## 一、可变参数

### 1.1 什么是可变参数

可变参数（Varargs）是 JDK 5 引入的特性，允许方法接收**任意数量**的同类型参数。

```java
public static int getSum(int... nums) {
    int sum = 0;
    for (int num : nums) {
        sum += num;
    }
    return sum;
}

// 调用：可以传任意个数
System.out.println(getSum(1, 2));             // 3
System.out.println(getSum(1, 2, 3, 4, 5));   // 15
System.out.println(getSum());                 // 0（不传也可以）
```

### 1.2 语法规则

| 规则 | 说明 |
|------|------|
| 格式 | `数据类型... 变量名` |
| 本质 | 底层是数组，编译后就是数组参数 |
| 位置 | **必须放在参数列表最后**，一个方法只能有一个可变参数 |
| 传值 | 可以传数组，也可以直接传零散数据 |

```java
// 正确：位于最后
public void method(String name, int... scores) { }

// 错误：可变参数必须在最后
// public void method(int... scores, String name) { }

// 错误：只能有一个可变参数
// public void method(int... a, String... b) { }
```

### 1.3 应用场景

方法不确定接收多少个参数时使用，比如：

```java
// 字符串拼接
public static String join(String separator, String... strs) {
    return String.join(separator, strs);
}
```

## 二、Collections 工具类

### 2.1 概述

`java.util.Collections` 是操作集合（List、Set、Map 等）的**工具类**，全是静态方法。

### 2.2 常用方法

| 方法 | 说明 |
|------|------|
| `addAll(Collection, T...)` | 批量添加元素 |
| `shuffle(List)` | 打乱 List 顺序（随机排列） |
| `sort(List)` | 排序（自然顺序，元素需实现 Comparable） |
| `sort(List, Comparator)` | 按比较器排序 |
| `binarySearch(List, key)` | 二分查找（需先排序） |
| `copy(List dest, List src)` | 复制（dest.size() >= src.size()） |
| `fill(List, Object)` | 填充所有元素 |
| `max/min(Collection)` | 求最值 |
| `swap(List, i, j)` | 交换两个位置 |
| `reverse(List)` | 反转顺序 |

```java
List<String> list = new ArrayList<>();

// 批量添加
Collections.addAll(list, "张三", "李四", "王五", "赵六");
System.out.println(list);     // [张三, 李四, 王五, 赵六]

// 打乱顺序
Collections.shuffle(list);
System.out.println(list);     // [王五, 张三, 赵六, 李四]（随机）

// 排序
Collections.sort(list);
System.out.println(list);     // [张三, 李四, 王五, 赵六]（自然顺序）

// 自定义排序
Collections.sort(list, (a, b) -> b.compareTo(a));   // 降序
```

### 2.3 shuffle 底层原理

`shuffle` 使用 Fisher–Yates 随机洗牌算法：

```
从最后一个元素开始，随机选取一个位置交换，依次向前
```

也是后续斗地主项目中洗牌的核心逻辑。

## 三、综合练习 1~3：随机点名器

### 3.1 练习 1：基础随机点名

```java
// 需求：从集合中随机获取一个学生姓名
ArrayList<String> students = new ArrayList<>();
Collections.addAll(students, "小明", "小红", "小刚", "小丽", "小华", "小美");

// 方式一：Random
Random r = new Random();
int index = r.nextInt(students.size());
System.out.println(students.get(index));

// 方式二：Collections.shuffle + 取第一个
Collections.shuffle(students);
System.out.println(students.get(0));
```

**两种方式对比：**
| 方式 | 特点 |
|------|------|
| `Random.nextInt(size)` | 纯随机，每次独立 |
| `shuffle + get(0)` | 打乱后取第一个，适合连续点名不重复的场景 |

### 3.2 练习 2：带概率的随机点名

**需求：** 女生被点到的概率是男生的两倍。

```java
ArrayList<String> boys = new ArrayList<>();
ArrayList<String> girls = new ArrayList<>();
Collections.addAll(boys, "小明", "小刚", "小华", "小强");
Collections.addAll(girls, "小红", "小丽", "小美");

// 思路：往面试名单中男生加1份，女生加2份
ArrayList<Integer> weightList = new ArrayList<>();
Collections.addAll(weightList, 1, 1, 1, 1, 2, 2, 2);

// 先抽权重
Random r = new Random();
int weightIndex = r.nextInt(weightList.size());
int weight = weightList.get(weightIndex);

// 根据权重从对应性别中抽人
if (weight == 1) {
    System.out.println(boys.get(r.nextInt(boys.size())));
} else {
    System.out.println(girls.get(r.nextInt(girls.size())));
}
```

**另一种思路：** 按人数成比例放大，将所有人按性别权重放入一个大集合再抽。

### 3.3 练习 3：不重复的随机点名

**需求：** 全班 80 人，点过的名字不再出现，直到所有人都被点过至少一轮。

**核心思路：** 维护一个"已点名"集合，每次从剩余未点名的集合中随机抽取。

```java
ArrayList<String> allStudents = new ArrayList<>();   // 全体（不变）
ArrayList<String> remaining = new ArrayList<>();     // 剩余（动态变化）

// 初始化
Collections.addAll(allStudents, /* 80 个学生姓名 */);
remaining.addAll(allStudents);

Random r = new Random();

// 点名一次
int index = r.nextInt(remaining.size());
System.out.println("点到：" + remaining.get(index));
remaining.remove(index);

// 所有人都点过一遍后重置
if (remaining.isEmpty()) {
    remaining.addAll(allStudents);
    System.out.println("=== 一轮结束，重新开始 ===");
}
```

**需要注意的点：**
- 每次点名后从剩余列表中删除该元素
- 剩余列表为空时重置
- 用 `ArrayList` 的随机访问 O(1) + 删除 O(n)

## 四、综合练习 4~5：带权重的随机点名

### 4.1 需求分析

每个学生有不同的权重，权重越高被点到的概率越大。

```java
// 用 Map 存储：姓名 → 权重
Map<String, Integer> weightMap = new HashMap<>();
weightMap.put("小明", 1);
weightMap.put("小红", 2);   // 权重更高
weightMap.put("小刚", 1);
weightMap.put("小丽", 3);   // 权重最高
```

### 4.2 实现思路

**加权随机选择算法：**

```
1. 计算总权重 totalWeight
2. 生成 [0, totalWeight) 的随机数
3. 遍历权重列表，累加权重，当累加值 > 随机数时选中
```

```java
// 加权随机选择
public static String weightedRandom(Map<String, Integer> weightMap) {
    // 计算总权重
    int totalWeight = weightMap.values().stream()
        .mapToInt(Integer::intValue).sum();
    
    // 生成随机数
    int random = new Random().nextInt(totalWeight);
    
    // 累加判断
    int cumulative = 0;
    for (Map.Entry<String, Integer> entry : weightMap.entrySet()) {
        cumulative += entry.getValue();
        if (cumulative > random) {
            return entry.getKey();
        }
    }
    return null;
}
```

**算法原理：** 想象一条线段，每个人占一段。随机指针落在哪段就选谁——权重越高线段越长，命中的概率越大。

## 五、综合练习 6：集合嵌套

### 5.1 什么是集合嵌套

集合的值本身也是集合，即**集合中存集合**。

### 5.2 案例：省份 → 城市

```java
// 数据结构：省份 → 多城市
Map<String, ArrayList<String>> provinceMap = new HashMap<>();

// 江苏省
ArrayList<String> jiangsu = new ArrayList<>();
Collections.addAll(jiangsu, "南京", "苏州", "无锡", "常州");
provinceMap.put("江苏省", jiangsu);

// 浙江省
ArrayList<String> zhejiang = new ArrayList<>();
Collections.addAll(zhejiang, "杭州", "宁波", "温州", "嘉兴");
provinceMap.put("浙江省", zhejiang);

// 广东省
ArrayList<String> guangdong = new ArrayList<>();
Collections.addAll(guangdong, "广州", "深圳", "东莞", "佛山");
provinceMap.put("广东省", guangdong);

// 遍历
for (Map.Entry<String, ArrayList<String>> entry : provinceMap.entrySet()) {
    System.out.println(entry.getKey() + "：" + String.join(", ", entry.getValue()));
}
// 江苏省：南京, 苏州, 无锡, 常州
// 浙江省：杭州, 宁波, 温州, 嘉兴
// 广东省：广州, 深圳, 东莞, 佛山
```

### 5.3 泛型嵌套写法

```
集合嵌套 = 泛型的泛型
┌───────────────────────────┐
│ Map<String, List<String>>  │
│ └─ 键类型 ─ 值类型 ─┘     │
│              ↓             │
│          List<String>      │
│          └─ 又是一个集合 ─┘│
└───────────────────────────┘
```

## 总结

| 知识点 | 核心要点 |
|--------|----------|
| 可变参数 | `类型... 变量名`，本质数组，**必须在参数最后** |
| Collections | 批量添加、打乱、排序、查找等静态工具方法 |
| 随机点名器 | 基础随机、概率加权、不重复轮次三种实现 |
| 加权随机算法 | 算总权重 → 随机数 → 累加判断 |
| 集合嵌套 | Map 的值存 List/Map，灵活的层次结构 |

**面试扩展：** 加权随机在**负载均衡**（Nginx 的加权轮询）、**抽奖系统**中广泛应用。数据量小时用累加法，数据量大时可改用**前缀和 + 二分查找**优化到 O(log n)。
