# 06 - Java 运行原理

## 1. 编译与运行

### 1.1 完整过程

```
┌──────────────────────────────────────────────────────────────────────┐
│  编译阶段                                        运行阶段              │
│                                                                      │
│  Hello.java ──→ [javac] ──→ Hello.class ──→ [java] ──→ 控制台输出     │
│    (源码)        (编译器)       (字节码)        (JVM)                  │
│                                                                      │
│  ┌────────┐                  ┌────────┐                              │
│  │ javac  │                  │  java  │                              │
│  └────────┘                  └────────┘                              │
│  将 .java 编译成 .class      启动 JVM，加载并执行 .class                 │
│  检查语法错误                 解释执行 / JIT 即时编译为机器码              │
│                              内存管理 (GC)                             │
└──────────────────────────────────────────────────────────────────────┘
```

### 1.2 编译阶段详解

```bash
# javac 做了什么
javac HelloWorld.java

# 1. 词法分析：字符序列 → Token 流
# 2. 语法分析：Token 流 → 抽象语法树 (AST)
# 3. 语义分析：类型检查、变量检查等
# 4. 生成字节码：AST → HelloWorld.class（.class 字节码文件）
```

```bash
# 反编译查看字节码
javap -c HelloWorld.class
```

### 1.3 运行阶段详解

```
java HelloWorld

1. 类加载：ClassLoader 加载 HelloWorld.class 到 JVM
2. 字节码校验：验证 .class 文件格式正确性
3. 解释执行：JVM 逐条将字节码翻译为机器码执行
4. JIT 编译：热点代码（频繁执行）直接编译成本地机器码，提升性能
5. GC：自动回收不再使用的内存
```

## 2. JVM 内存模型

### 2.1 JVM 内存分区

```
┌──────────────────────────────────────────────────────────────┐
│                         JVM 内存                              │
│                                                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐        │
│  │   栈 (Stack)  │  │  堆 (Heap)   │  │ 方法区        │        │
│  │              │  │              │  │ (Method Area) │        │
│  │ 每个线程独立  │  │ 所有线程共享  │  │              │        │
│  │              │  │              │  │ 所有线程共享  │        │
│  └──────────────┘  └──────────────┘  └──────────────┘        │
│                                                              │
│  ┌──────┐ ┌──────┐ ┌──────────────────────┐                  │
│  │ 程序  │ │ 本地  │ │ 直接内存              │                  │
│  │ 计数器 │ │ 方法栈 │ │ (元空间/堆外内存)      │                  │
│  └──────┘ └──────┘ └──────────────────────┘                  │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 各区域详解

| 区域 | 存储内容 | 线程 | 特点 |
|------|----------|------|------|
| **栈 (Stack)** | 局部变量、方法参数、方法调用链 | **线程私有** | 方法调用进栈，结束出栈；容量小 (~1MB)，用完自动释放 |
| **堆 (Heap)** | `new` 出来的对象、数组实例 | **线程共享** | GC 主要区域，容量大，对象没有引用时被回收 |
| **方法区** | 类信息、静态变量、常量池、方法字节码 | **线程共享** | JDK 8+ 用**元空间**（本地内存）替代永久代 |
| **程序计数器** | 当前线程执行的字节码行号 | **线程私有** | 极小，记录"执行到哪了" |
| **本地方法栈** | `native` 方法调用 | **线程私有** | C/C++ 写的本地方法 |

### 2.3 方法执行的内存过程

```java
public class Demo {
    public static void main(String[] args) {
        int[] arr = new int[]{10, 20, 30};   // 第 4 行
        int max = getMax(arr);                // 第 5 行
        System.out.println(max);
    }

    public static int getMax(int[] array) {
        int max = array[0];                   // 第 10 行
        return max;
    }
}
```

```
执行过程：

1. 类加载：Demo.class 加载到方法区
2. main 方法被调用 → main 栈帧入栈
3. new int[] → 在堆中创建数组对象 {10,20,30}
4. arr 变量存储在 main 栈帧中，指向堆中的数组地址
5. getMax(arr) → getMax 栈帧入栈，压入 main 之上
6. array 变量存储在 getMax 栈帧中，指向同一个堆地址（地址拷贝）
7. getMax 执行完毕 → getMax 栈帧出栈（销毁局部变量）
8. main 继续执行 println → main 栈帧出栈 → 程序结束
```

## 3. 类加载机制

### 3.1 类加载的三个阶段

```
加载 (Loading) → 连接 (Linking) → 初始化 (Initialization)
                       │
            ┌──────────┼──────────┐
            │          │          │
            ▼          ▼          ▼
           验证        准备        解析
```

| 阶段 | 做什么 |
|------|--------|
| **加载** | 读取 `.class` 文件，生成 `Class` 对象 |
| **验证** | 检查字节码格式、安全性（防止恶意代码） |
| **准备** | 为**静态变量**分配内存，赋默认值 |
| **解析** | 符号引用 → 直接引用（如 `String` → 实际内存地址） |
| **初始化** | 执行静态变量赋值和静态代码块 |

### 3.2 类加载器层次（双亲委派模型）

```
┌──────────────────────────┐
│  Bootstrap ClassLoader    │  ← 加载核心类库 (rt.jar → java.base)
│  (启动类加载器，C++ 实现)   │
└──────────┬───────────────┘
           │ 父
           ▼
┌──────────────────────────┐
│  Extension ClassLoader    │  ← 加载扩展类库 (jre/lib/ext)
│  (扩展类加载器)            │     JDK 9+ 被 Platform ClassLoader 替代
└──────────┬───────────────┘
           │ 父
           ▼
┌──────────────────────────┐
│  Application ClassLoader  │  ← 加载 classpath 下的自定义类
│  (应用类加载器)             │
└──────────────────────────┘
```

### 3.3 双亲委派机制

```
加载请求 ──→ App ClassLoader → Ext ClassLoader → Bootstrap ClassLoader
                                                       │
                                           ┌───────────┘
                                           ▼
                                    能找到？← 是 → 加载
                                           │
                                           ▼ 否
                                    向下委派给子加载器
                                           │
                                           ▼
                                    都找不到 → ClassNotFoundException
```

**为什么要双亲委派？**
- 安全性：核心类（如 `java.lang.String`）只能由 Bootstrap 加载，防止被恶意替换
- 避免重复加载：父加载器已加载的类，子加载器不再加载

## 4. 垃圾回收 (GC) 简介

### 4.1 什么是 GC？

Garbage Collection — JVM 自动回收**堆内存中不再被引用的对象**。

```java
// C/C++ 手动管理
int* p = malloc(sizeof(int) * 100);
// ... 使用 ...
free(p);  // 必须手动释放，忘了就内存泄漏

// Java 自动管理
int[] arr = new int[100];
arr = null;      // 失去引用
// JVM 会在适当的时机自动回收 arr 原来指向的内存
```

### 4.2 堆内存分代

```
┌───────────────────────────────────────────────┐
│                   堆内存                        │
│  ┌─────────────────────┐                      │
│  │     新生代 (Young)    │  ← Minor GC 频繁      │
│  │  ┌────┬────┬───────┐ │                      │
│  │  │Eden│ S0 │  S1   │ │  新对象在 Eden 创建    │
│  │  │    │    │       │ │  存活 → Survivor      │
│  │  └────┴────┴───────┘ │  多次存活 → 老年代     │
│  └─────────────────────┘                      │
│  ┌─────────────────────┐                      │
│  │    老年代 (Old)      │  ← Major GC 较慢      │
│  │                     │  大对象直接进入老年代    │
│  └─────────────────────┘                      │
└───────────────────────────────────────────────┘
```

| 区域 | 说明 |
|------|------|
| **Eden** | 新对象默认分配区，大部分对象"朝生夕死" |
| **Survivor (S0/S1)** | Eden 中经过一次 GC 存活的对象进入 Survivor |
| **Old** | 经过多次 GC 仍存活的对象晋升到老年代 |
| **MetaSpace (元空间)** | JDK 8+ 替代永久代，存类元数据，使用本地内存 |

### 4.3 GC 算法简介

| 算法 | 思路 | 特点 |
|------|------|------|
| **标记-清除** | 标记垃圾 → 清除 | 简单但会产生内存碎片 |
| **标记-整理** | 标记垃圾 → 存活对象向一端移动 | 无碎片，但移动成本高 |
| **复制算法** | 内存分两块，存活对象复制到另一块 | 无碎片、效率高，浪费一半空间（新生代使用） |
| **分代收集** | 新生代用复制算法，老年代用标记-整理 | **JVM 实际采用的组合策略** |

## 5. 总结

| 知识点 | 核心要点 |
|--------|----------|
| 编译阶段 | `javac` → 词法分析 → 语法分析 → 语义分析 → 生成 `.class` |
| 运行阶段 | 类加载 → 字节码校验 → 解释执行 + JIT 编译 → GC 内存管理 |
| 栈 | 线程私有，存局部变量和方法调用，方法结束自动释放 |
| 堆 | 线程共享，存 `new` 出来的对象，GC 主要管理区域 |
| 方法区 | 线程共享，存类信息、静态变量、常量池（JDK 8+ 元空间） |
| 类加载 | 加载 → 连接(验证+准备+解析) → 初始化 |
| 双亲委派 | 向上查找，向下加载；安全 + 避免重复加载 |
| GC | JVM 自动回收不再引用的堆对象，无需手动 `free` |
| 分代回收 | 新生代 (Eden+S0+S1) 用复制算法，老年代用标记-整理 |
| JIT 编译 | 热点代码直接编译为机器码，提升执行速度 |
