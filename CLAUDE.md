## 笔记编写规范

1. **文件命名**：`XX-主题.md`，XX 为两位数序号，按知识板块排列（非按学习天数）
2. **模块文件夹命名**：按技术栈分文件夹，名称简洁明了（如 `redis缓存`、`JavaSE`、`javaweb`）
3. **内容结构**：问题驱动（场景 → 问题 → 原因 → 方案 → 代码 → 新问题 → 继续优化），末尾附总结表格
4. **风格要求**：
   - GitHub 风格表格、代码块标注语言类型（java/bash/lua/sql/javascript/vue）
   - ASCII 架构图/流程图
   - 代码片段以 Java 为主（Spring Boot + MyBatis），前端部分用 JS/Vue
   - 核心知识点用表格对比（如 Filter vs Interceptor、`#{}` vs `${}`）
5. **新增笔记时**：同步更新 `README.md` 目录表格，确保链接正确

## 典型任务

当用户说"今天学了xxx，帮我写笔记"时：
1. 确认当天学习的具体知识点和顺序
2. 按照已有笔记的风格撰写（参考最新笔记的格式）
3. 写完后同步更新 README.md

当用户说"温故/复习了xxx"时（如 Java SE 复习笔记）：
1. 确认复习的具体章节和知识点
2. 在新文件夹下按章节编号创建笔记，风格保持一致（问题驱动 + 代码 + 总结表格）
3. 同步更新 README.md

当用户给出已有讲义要求整理笔记时：
1. 先浏览讲义结构（用 Grep 提取所有标题），了解章节划分和知识点
2. 按知识板块合并相关章节，合并策略：
   - 同主题多章合并（如 08/09/10 员工管理 → 2 篇）
   - 相关小章合并（如 Maven 基础 + 高级 → 1 篇）
   - 纯练习章跳过（如 11-自己完成，无新知识点）
3. 提炼核心知识点，精简原讲义中的广告/冗余图片/飞书链接
4. 保持统一的笔记风格（代码示例 + 总结表格）
5. 写完后同步更新 README.md
